# v2.0.0 — 重大重构

**发布日期**: 2026年3月9日  
**类型**: 核心版本

---

## 破坏性变更

### KVCache API 重新设计

**修复的关键 Bug**: 层序依赖的写入位置问题

之前的 `appendKV()` 存在脆弱的层序依赖：
- Layer 0 会更新 `current_len`，其他层尝试补偿
- 如果层调用顺序不同，逻辑会崩溃
- 可能产生负的 `write_pos`

**新 API**:

1. `appendKV()` 现在是**无状态的**
   - 所有层都写入 `current_len` 位置
   - 不依赖层索引

2. 新增 `advanceSeqLen()` 方法
   - 所有层完成 append 后显式调用一次
   - 清晰的职责分离：append 是每层操作，长度更新是每步操作

```cpp
// 修复后
for (auto &layer : layers_) {
  layer->forward(hidden_states, *kv_cache_, seq_id, position, stream_);
}
kv_cache_->advanceSeqLen(seq_id, 1);  // 显式更新
```

详细设计请参考 [架构设计](../../docs/zh/ARCHITECTURE)。

## 变更内容

### CMake 现代化

| 变更 | 之前 | 之后 |
|------|------|------|
| 版本 | 1.0.0 | 2.0.0 |
| CUDA 架构 | 手动 | 自动检测 (native 或回退) |
| Include | 全局 | `target_include_directories()` |
| Target | 单个 | 新增 `tiny_llm::tiny_llm` 别名 |
| 警告 | 无 | `-Wall -Wextra` (GCC/Clang) |
| IDE | 基础 | `compile_commands.json` |

## 迁移指南

如果直接使用 KVCache，请更新代码：

```cpp
// 旧代码 (v1.x)
kv_cache.appendKV(seq_id, layer_idx, k, v, 1);

// 新代码 (v2.0+)
kv_cache.appendKV(seq_id, layer_idx, k, v, 1);
// 所有层完成后...
kv_cache.advanceSeqLen(seq_id, 1);
```
