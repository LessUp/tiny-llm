# v2.0.0 — Major Refactoring

**Release Date**: March 9, 2026  
**Type**: Core Release

---

## Breaking Changes

### KVCache API Redesign

**Critical Bug Fixed**: Layer-order-dependent write position bug

The previous `appendKV()` had fragile layer-order dependencies:
- Layer 0 would update `current_len`, other layers tried to compensate
- Logic would break if layer call order differed
- Could produce negative `write_pos`

**New API**:

1. `appendKV()` is now **stateless**
   - All layers write to `current_len` position
   - No dependency on layer index

2. New `advanceSeqLen()` method
   - Explicitly call once after all layers complete append
   - Clear separation: append is per-layer, length update is per-step

```cpp
// After fix
for (auto &layer : layers_) {
  layer->forward(hidden_states, *kv_cache_, seq_id, position, stream_);
}
kv_cache_->advanceSeqLen(seq_id, 1);  // Explicit update
```

See [Architecture](../../docs/en/ARCHITECTURE) for detailed design.

## Migration Guide

Update your code if using KVCache directly:

```cpp
// Old code (v1.x)
kv_cache.appendKV(seq_id, layer_idx, k, v, 1);

// New code (v2.0+)
kv_cache.appendKV(seq_id, layer_idx, k, v, 1);
// After all layers...
kv_cache.advanceSeqLen(seq_id, 1);
```
