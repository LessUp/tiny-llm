# v2.0.2

**Release Date:** 2026-04-27

## Summary

Code quality improvements, GGUF quantization utilities, and repository hardening.

## Changes

### Code Quality

- **KVCacheManager**: Changed constructor to factory method `create()` for consistent `Result<T>` error handling
- **noexcept**: Added `noexcept` specifier to simple accessor methods
- **F32→F16 conversion**: Extracted to utility function `convertF32ToF16()` in new `quantization.h`
- **Code formatting**: Fixed clang-format-18 violations in quantization.cpp, kv_cache.cpp, model_loader.cpp
- **main.cpp**: Added `--help`, `--version`, `--info` CLI options for better usability

### Infrastructure

- **CI simplification**: Replaced complex CUDA fallback logic with `Jimver/cuda-toolkit` action
- **LSP configuration**: Enhanced `.clangd` with fallback compile flags for better IDE support
- **AI tooling**: Added core architecture documentation to CLAUDE.md and AGENTS.md

### New Features

- **Quantization utilities**: Added `quantization.h`/`quantization.cpp` with:
  - `convertF32ToF16()` - F32 to F16 conversion
  - `dequantizeQ4_0()` - Q4_0 block dequantization
  - `dequantizeQ8_0()` - Q8_0 block dequantization
  - `quantizeF16ToW8A16()` - F16 to W8A16 quantization

### Repository Cleanup

- Removed empty `docs/` directory
- Removed deprecated `_bmad/` and `_bmad-output/` directories
- Removed `.qwen/` configuration
- Archived `normalize-archive-readiness` change
- Reduced `.claude/skills/` from 46 skills (238 files) to 5 core skills

### Documentation

- Updated copyright year to 2026
- Updated KVCacheManager usage examples in API documentation
- Added Chinese and English changelog for v2.0.2
- Added core architecture table to CLAUDE.md and AGENTS.md

## Compatibility

- No breaking API changes
- `KVCacheManager` constructor is now private; use `KVCacheManager::create()` factory method

## Upgrade Guide

```cpp
// Old (v2.0.1)
KVCacheManager cache(config);

// New (v2.0.2)
auto cache_result = KVCacheManager::create(config);
if (cache_result.isErr()) {
    // Handle error
}
auto cache = std::move(cache_result.value());
```
